Languages Support

There are so many programming languages around these days that it will be impossible to cater for all of them. The way Colorizor was programmed made it easy for anyone to cater for their own language.

When using custom templates Use the colorizor.dev.js file. This will not include all the feature, like auto theme, auto language, ...

Setup

Setting up your own language is very quick and easy. You would need to know a bit about Regular Expressions to find what you are looking for in you code.
In order to setup your language files, you'll need to host your own styling and script files. The processing and functions are all handled by the colorizor.dev.js script.
Lets get started...


HTML

<script type="text/javascript" src="https://colorizor.github.io/colorizor.dev.js" async></script>
<link rel="stylesheet" type="text/css" href="https://colorizor.github.io/Test/style.css" />

This is what you'll need to add to the head of your markdown file.


Creating Your Language

There is a basic structure for creating your own language. It consists of four main parts: language, prepare, execute and finalise.
The structure is in a form of a dictionary. The language and execute are recommended keys to have in this dictionary.

(function() {
  const data = {
    language: 'none',
    prepare: [],
    execute: [],
    finalise: []
  };
  clz.Colorize(data);
})();

You will only need to make changes to: language, prepare, execute and finalise
The language key will determine for what language this script is for, e.g. javascript = The JavaScript language.


Prepare & Finalise

These two keys will mainly be used for formatting special occurances of some sort in the language defined.
e.g. In some langauges the // will be used to comment something, well in a url, http://... it also contains it. To make life easier we can format all url's to replace :// with :/\/. When the main execution has completed the finalise is called and there we can once again fix the problem by replacing the newly formatted url's, :\/\ back to its original state, ://.

(function() {
  const data = {
    language: 'none',
    prepare: [
      {
        pat: /\:\/\//gm,   //Pattern
        rep: '\:\/\\\/'    //Replace
      }
    ],
    execute: [],
    finalise: [
      {
        pat: /\:\/\\\//gm,  //Pattern
        rep: '\:\/\/'       //Replace
      }
    ]
  };
  clz.Colorize(data);
})();


Execute

There are three types of execution techniques to choose from:

Technique 1

This technique is handy for narrowing the results down to something between objects.

{
  begin: {
    pat: '',          //String Based RegEx
    exclude: true     //Exclude this begining?
  },
  end: {
    pat: '',          //String Based RegEx
    exclude: true     //Exclude this end?
  },
  pat: //gm,          //Pattern to find within the search results of the begin and end
  rep: ''             //Replace with
}

Technique 2

This technique is used for more of a global result.

{
  custom: '',         //String Based RegEx
  pat: //gm,          //Pattern to find within the search results
  rep: ''             //Replace with
}

Technique 3

This is very helpful when just searching for keywords or stuff of the same sort.

{
  keywork: //gm,      //Pattern
  rep: ''             //Replace with
}

Example

HTML

<head>
  <script type="text/javascript" src="https://colorizor.github.io/colorizor.dev.js" async></script>
  <link rel="stylesheet" type="text/css" href="https://colorizor.github.io/Test/style.css" />
</head>
<body>
  <pre><code language="colorize" file="https://colorizor.github.io/Test/colorize.js">
    :: This language, Colorize was created by Colorizor
    String:Text > 'Hello, This is a custom language'
    PrintString():Execute > (
      :Print > String
    )
  </code></pre>
</body>

CSS

span#comment {
  color: rgb(159, 253, 193);
  font-style: italic;
}
span#keywork {
  color: rgb(255, 127, 102);
}
span#text {
  color: rgb(126, 206, 253);
}

Output

:: This language, Colorize was created by Colorizor

PrintString():Execute > (
  :Print > String
